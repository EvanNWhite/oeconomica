# Data Visualization

**Level:** Beginner–Intermediate
**Prerequisites:** `01-data-cleaning`, ideally `02-data-analysis`

## Why this matters

A policy memo lives or dies on whether the reader can see the pattern
you're claiming exists. Pick the chart type based on what you're showing,
not on what looks fanciest.

## Which chart for which question

| Question | Chart |
|---|---|
| How has X changed over time? | Line chart |
| Is there a relationship between two variables? | Scatterplot |
| How do a few categories compare at one point in time? | Bar chart |
| What's the spread/shape of one variable? | Histogram |
| How do distributions compare across groups? | Boxplot |

## Guidelines

- Always label axes and give the chart a title — a chart with no label
  meaning is not usable in a memo
- Don't put more than ~4-5 lines/categories on one chart, or it becomes
  unreadable
- If you're comparing early vs. late periods (see `02-data-analysis`), a
  simple line chart with the two averages marked, or a bar chart of a few
  selected years, communicates the same point more directly than showing
  every single data point
